    • 10. what is the flow of git???

    The typical Git workflow involves the following steps:

  • 1. *Initialize Repository*: Create a local repository using git init or clone a remote one with git clone.
  • 2. *Make Changes*: Edit files or add new ones in the working directory.
  • 3. *Stage Changes*: Use git add file to move changes to the staging area.,
  • 4. *Commit Changes*: Save a snapshot of staged changes to the local repository using `git commit -m "message"`.
  • 5. *Push Changes*: Upload commits from the local repository to a remote repository with git push.
  • 6. *Pull/Fetch Changes*: Retrieve updates from the remote repository using git pull or git fetch.
  • 7. *Merge*: Combine changes, if necessary, to resolve conflicts between branches or collaborators.
    • Optional steps:
  • - *Branching*: Create new branches for feature development with git branch branch-name.
  • - *Merging*: Merge branches into the main project using git merge.
    • This process enables efficient collaboration and version control.
